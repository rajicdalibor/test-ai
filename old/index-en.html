Use Case: Claims Processing

Processing claims through the AI in a Box platform enables health insurance companies to increase the efficiency and customer satisfaction. Partnering with Adcubum, we fully integrate the E-Claims Plus module of Adcubum Syrius.

The platform is capable of classifying TARMED invoices, pharmacy receipts as well as optician and fitness invoices, extracting their content and input them into Adcubum Syrius applying the insurance's specific business rules. This allows an insurance company to have an end to end and fully automated claims process.

The sources of the invoices can be various and individual for every insurance. The platform can either be plugged into an existing process using OpenAPIs or new inbound queues can be created.

Inbound Queue

Companies receive data in a wide variety of ways and forms. Using OpenAPI, the AI in a Box platform can handle any kind of inbound data.

Quality Check

Because data is received in various ways, we need to make sure the quality of the data is sufficient for further processing. If bad quality data is detected, immediate feedback can be given to make sure the process can be fully automated in the next steps.

Document Classification

Using a ML algorithm, we determine the type of the document / invoice. Supervising this process step using our platform UI ensures continuous improvement and improved model accuracy.

Extraction

To get structured data out of unstructured documents we use an enhanced extraction process to understand the content of the invoice.

Content Classification

To understand the content of the now structured data, we use NLP to get the meaning of the content. Supervising this process step using our platform UI ensures continuous improvement and improved model accuracy.

Enrichment

The extracted and classified data can be enriched with information from external systems like a policy number from an insurance core system.

Business Rule Application

The classified data can now be used and mapped to any business logic.
